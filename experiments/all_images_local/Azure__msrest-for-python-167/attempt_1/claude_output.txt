## Summary

I've successfully fixed the issue. The problem was that the content type JSON parser only supported exact matches for `"application/json"` and `"text/json"`, but according to the HTTP spec, `"application/<text>+json"` formats (like `application/ld+json`, `application/vnd.api+json`, `application/hal+json`) should also be accepted.

### Changes Made:

1. **Added `_is_json_content_type()` method** in `/testbed/msrest/pipeline/universal.py` that:
   - Checks for exact matches from `JSON_MIMETYPES`
   - Checks for `application/*+json` and `text/*+json` formats
   - Returns `True` for valid JSON content types

2. **Updated `deserialize_from_text()`** to use the new helper method instead of direct list membership

3. **Added tests** for common application/*+json formats in `/testbed/tests/test_universal_pipeline.py`
