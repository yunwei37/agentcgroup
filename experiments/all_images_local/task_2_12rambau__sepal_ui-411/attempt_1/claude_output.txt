## Summary

I've successfully added a `disabled` trait to the `DatePicker` widget. The implementation allows users to easily enable/disable the datepicker by setting `self.disabled = True/False` instead of manually accessing the menu and slots.

### Changes Made:

1. **`/testbed/sepal_ui/sepalwidgets/inputs.py`**:
   - Added `Bool` to imports
   - Added `disabled` trait as a class attribute: `disabled = Bool(False).tag(sync=True)`
   - Modified `__init__` to accept and set the `disabled` parameter
   - Added `@observe("disabled")` decorator to monitor trait changes
   - Implemented `_on_disabled_change()` method to update both the menu and slots

2. **`/testbed/tests/test_DatePicker.py`**:
   - Added `test_disabled()` method to verify the functionality

### Usage:

**Before:**
```python
self.w_start = sw.DatePicker(label="start", v_model=None)
self.w_start.menu.v_slots[0]["children"].disabled = True
self.w_start.menu.disabled = True
```

**After:**
```python
self.w_start = sw.DatePicker(label="start", v_model=None, disabled=True)
# or
self.w_start.disabled = True
```

The code compiles successfully without syntax errors and follows the existing pattern used in other widgets (like `Btn`).
